# Chinese Casual Phrases (Default 活人感词库)

默认的中文口语词库——给没有配置 `preferred_phrases` 的新用户一个"活人感底色"。Phase 3 写稿时，skill 会自然穿插这些词组，让文章不会"AI 腔过重"。

**这不是强制**。用户可以在 EXTEND.md 里：
- 通过 `preferred_phrases` 添加自己的偏好词 → 和本词库并集
- 通过 `disable_default_casual_phrases: true` 完全关掉本词库
- 通过 `banned_phrases` 禁掉本词库里某些词

## 使用原则

- **自然穿插，不堆砌**。一篇 2000 字文章默认用 5-10 个词组；**严格活人感模式下（EXTEND.md `human_voice_strict: true`）提升到 8-12 个**
- **位置决定效果**：
  - 转场词放段落开头或段落结尾拉主线
  - 判断词放观点句之前，降低 AI 腔
  - 情绪词放金句附近，增加温度
  - 自嘲词放敏感观点之前，卸掉"我来教你"的姿态

## 反向清单：默认禁用

这些是 AI 写作里的高频"AI 味"词，skill 默认要避开：

| 禁用 | 替代 |
|---|---|
| 说白了 | 坦率地讲 / 其实就是 |
| 意味着什么 | 那会怎样 / 所以呢 |
| 这意味着 | 也就是说 / 你想想看 |
| 本质上 | 说到底 / 其实 |
| 换句话说 | 你想想看 / 也就是说 |
| 不可否认 | 直接删 / 正面陈述 |
| 综上所述 | 回到前面说的 / 简单说 |
| 总的来说 | 我的判断是 / 所以 |
| 首先...其次...最后 | 用自然转场词（"先说第一点" / "再看这个" / "最后" ）|
| 值得注意的是 | 直接讲 |
| 不难发现 | 直接讲 |
| 让我们来看看 | 直接讲 |
| 接下来让我们 | 接着说 / 说到这个 |
| 在当今...的时代 | 直接从具体事件切入 |

## 在不同定位下的使用强度

| Positioning | 口语词组使用密度 | 说明 |
|---|---|---|
| 思辨文 | 中（5-8 个/2000 字） | 太多会削弱判断锋利度 |
| 攻略 | 高（10-15 个/4000 字） | 攻略最怕"教科书腔"，口语化是解药 |
| 随笔 | 高（8-12 个/1500 字） | 私人化基调，自嘲和情绪词多用 |
| 评论 | 中低（3-5 个/1000 字） | 评论要锐，不要太柔 |
| 翻译 | 低（1-3 个整篇） | 译文要贴近原文节奏 |

## Skill 使用指引

在 Phase 3 写稿时，按以下原则用：

1. **不要在每句话都用**。一段话最多 1 个，整篇平均每 200-300 字 1 个
2. **配合 EXTEND.md**：用户的 `preferred_phrases` 优先于本词库。本词库填补空白
3. **L2 风格检查**里会校验"至少用了 5 个不同的口语词组" ——如果新用户不配置，默认从本词库抽取
4. **不要让同一个词在一篇文章里出现 3 次以上**，会显得敷衍

## 禁用本词库的方式

如果你想完全接管默认风格：

```yaml
# EXTEND.md
disable_default_casual_phrases: true
preferred_phrases:
  - ... # 你自己的词库
```

这样 skill 只用你自己的 `preferred_phrases`，不碰默认词库。适合有明确个人腔调（比如学术风、冷峻风）的写作者。
